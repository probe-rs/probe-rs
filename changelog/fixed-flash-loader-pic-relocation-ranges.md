Fixed flash-loader PIC relocation handling for GCC-linked algorithms by deriving relocation metadata from ELF relocation records instead of guessing from `.got` sections, so probe-rs now rebases only the exact relocation slots instead of patching words heuristically. The flash algorithm metadata was tightened up by separating `static_base_offset` from `data_section_offset`, preserving `data_section_offset` as the `PrgData` offset, and normalizing the public YAML/schema representation to compact `address_relocation_ranges` that merge contiguous 4-byte relocation slots and split on holes. On the generation side, `target-gen` now emits relocation ranges directly from ELF relocation entries, drops PIC relocation metadata for fixed-load algorithms, and has end-to-end fixture coverage for both zero-VMA and nonzero-VMA PIC extraction paths. On the runtime side, flash algorithm loading now expands the compact ranges back into exact relocation offsets before rebasing, validates relocation bounds and link-time image membership, and keeps the improved `static_base_offset` fallback behavior covered by dedicated rebasing and error-path tests.
