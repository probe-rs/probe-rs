Changed the DAP server to emit DWARF source paths verbatim in stack frames, breakpoints, and disassembly instead of validating them against the server's filesystem and rewriting unresolvable paths to `<unavailable>: <name>`. Source-path resolution now lives entirely on the client side, where the user's source tree and toolchains are. The probe-rs VSCode extension applies a configurable `sourceFileMap` for language-agnostic prefix rewrites (e.g. CI build paths, cross-compile sysroots), and auto-populates the `/rustc/<hash>/...` → local sysroot rewrite for Rust users. This unbreaks source navigation for both local and remote `dap-server` setups, and adds a working knob for C/C++ users whose binaries record build-host paths.
