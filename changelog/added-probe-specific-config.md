The ability to configure options specific to each probe type has been added.  This is done programmatically by passing a ProbeSettings struct into a `open_with_settings` call or using the new `Lister::with_settings` API.  Via the CLI, the `.probe-rs.toml` or similar configuration file supports `probe_settings` keys. This is useful for probes on custom PCBs or generic probes with non-standard or additional wiring.  As of now, only the reset pin identifier for FTDI-based probes is configurable via this struct.
